# 🎮 Word Quest (워드 퀘스트)
> **게이미피케이션 기반 영단어 학습 RPG 시스템**

영단어 학습에 RPG의 '성장'과 '게임' 요소를 결합하여, 사용자가 몰입감 있게 단어를 외우고 복습할 수 있는 안드로이드 애플리케이션입니다.

---

## 🎯 프로젝트 목표
- **학습의 재미**: 스테이지 클리어 방식의 학습을 통한 성취감 고취.
- **성장 시스템**: 학습 결과로 얻은 장비를 합성하고 강화하여 전투력 상승.
- **체계적 복습**: 오답 노트를 기반으로 한 복습 시스템으로 학습 효율 극대화.

---

## 📱 화면 및 기능 구조
### 1. Start (시작/설정)
- 앱의 대문이며 시작하기 및 환경 설정(사운드, 진동, FPS 등) 기능 제공.
### 2. Learn (학습 + 스테이지)
- 난이도별/테마별 스테이지 기반 영단어 퀴즈 진행.
### 3. Review (오답 노트)
- 학습 중 틀린 단어를 자동으로 수집하고 재학습 기회 제공.
### 4. Game (미니게임)
- 학습한 단어를 활용하여 런게임 방식의 퀴즈 미니게임을 진행.

---

## 🛠 기술 스택 및 아키텍처
- **언어**: Kotlin (1.9.0+)
- **아키텍처**: MVVM (ViewModel + LiveData/Flow)
- **DB**: Room Persistence Library (Entity: User, Item, Word, Progress)
- **UI**: ViewBinding, ConstraintLayout (반응형), Material 3
- **애니메이션**: Lottie 또는 XML 기반 인터랙션 피드백

---

## ⚙️ 개발 및 협업 규칙 (Senior Developer 기준)
1. **ViewBinding 필수**: 모든 Layout 접근은 ViewBinding을 사용하며, 메모리 누수 방지를 위해 Fragment에서는 `_binding` 처리를 엄격히 준수한다.
2. **MVVM 구조**: UI 로직과 비즈니스 로직을 철저히 분리한다. Repository 패턴을 도입하여 데이터 소스를 캡슐화한다.
3. **Room DB 중심**: 모든 유저 데이터와 단어 데이터는 로컬 DB를 최우선으로 하며, 데이터 부재 시 'Fallback UI'를 반드시 제공한다.
4. **사용자 경험(UX)**:
   - 입력 누락 시 즉시 차단 및 안내.
   - 뒤로가기 버튼 클릭 시 진행 상황 유실 방지를 위한 경고 다이얼로그 표시.

---

## 📂 주요 소스 코드 구조
- `MainActivity.kt`: 시작 화면 및 설정 제어.
- `HomeActivity.kt`: 메인 네비게이션(3개 탭) 허브 액티비티.
- `Learn/Review/GameFragment.kt`: 각 기능별 독립적인 화면 단위.
- `data/`: Entity, DAO, Database 클래스 (Room).
- `viewmodel/`: 각 기능별 비즈니스 로직 처리.
